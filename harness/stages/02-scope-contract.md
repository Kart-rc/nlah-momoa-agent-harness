# Stage 02 — Scope Contract

## Owner

Scope Auditor

## Purpose

Convert the task into explicit acceptance criteria, boundaries, artifacts, validation gates, stop conditions, and risk assumptions.

## Inputs

- `TASK.md`
- `TASK_CLASSIFICATION.md`
- `AMBIGUITY_REGISTER.md`, if present

## Outputs

- `SCOPE_CONTRACT.md`
- `ACCEPTANCE_CRITERIA.md`

## Gate

The run cannot proceed to planning until required artifacts and validation gates are stated.

## Failure Modes

- incomplete acceptance criteria
- missing stop conditions
- missing validation tier
- unbounded scope

## Next Stage Rule

Proceed to planning when the scope contract is internally consistent and traceable to the task request.
