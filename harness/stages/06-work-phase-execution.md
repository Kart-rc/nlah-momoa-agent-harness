# Stage 06 — Work Phase Execution

## Owner

Implementer (or delegated child implementer)

## Purpose

Execute assigned work phases, produce artifacts, and collect implementation evidence while recording decisions and dissent.

## Inputs

- phase task contract
- referenced source files or repositories
- acceptance criteria and scope constraints

## Outputs

- work artifacts
- `IMPLEMENTATION_EVIDENCE.md` updates
- manifest updates for produced artifacts
- child response packets when delegated

## Gate

Phase is complete only when required artifacts exist and success criteria in the phase contract are met.

## Next Stage Rule

Proceed to integration when all planned phases reach terminal verdicts (`COMPLETE`, `COMPLETE_WITH_RISK`, or `BLOCKED` with escalation).
