# Stage 03 — Planning

## Owner

Planner

## Purpose

Create a staged execution plan with work phases, role assignments, artifacts, validation steps, and repair triggers.

## Inputs

- `SCOPE_CONTRACT.md`
- `ACCEPTANCE_CRITERIA.md`
- task-type harness files

## Outputs

- `PLAN.md`
- optional `WORK_PHASE_TASK.md` files for child workspaces

## Gate

The plan must map every acceptance criterion to at least one work item and one validation gate.

## Next Stage Rule

Proceed to plan review for Tier 1+ work. Tier 0 may proceed directly to execution.
