# Presentation Harness

## Purpose

Create audience-specific presentations with a clear narrative, decision objective, slide outline, speaker support, and validation for message quality.

## Presentation Types

- leadership deck
- pitch deck
- technical stakeholder deck
- roadmap deck
- architecture review deck
- incident review deck

## Required Inputs

- audience
- objective
- desired outcome
- presentation type
- depth
- format
- time limit, if known

## Required Artifacts

- `AUDIENCE_BRIEF.md`
- `MESSAGE_STRATEGY.md`
- `DECK_OUTLINE.md`
- `SLIDE_STORYBOARD.md`
- `SPEAKER_NOTES.md`
- `EXECUTIVE_SUMMARY.md`
- `RELEASE_PACKET.md`

## Workflow

1. Audience brief
2. Objective clarification
3. Message strategy
4. Narrative arc
5. Slide outline
6. Storyboard
7. Review
8. Final deck or markdown storyboard

## Validation Gates

- each slide has one clear message
- deck supports the stated decision or outcome
- audience concerns are addressed
- claims are evidence-backed or framed as assumptions
- story can be delivered in target time
