# Generic Documentation Harness

## Purpose

Deliver accurate, audience-aware documentation with traceable sources and usability checks.

## Required Inputs

- documentation objective
- audience and use context
- source materials or system context

## Required Artifacts

- `DOC_OBJECTIVE.md`
- `AUDIENCE_PROFILE.md`
- `SOURCE_INDEX.md`
- `DOC_DRAFT.md`
- `FACT_CHECK_REPORT.md`
- `STYLE_REVIEW.md`
- `RELEASE_PACKET.md`

## Workflow

1. Define objective and audience
2. Collect source material
3. Draft structure and content
4. Fact-check all claims
5. Review clarity, tone, and completeness
6. Revise and finalize

## Validation Gates

- claims are evidence-backed
- terminology is consistent
- instructions are executable
- known gaps are documented

## Role Model

- orchestrator: ensure audience and scope fit before drafting
- planner: break work into research, drafting, and review phases
- implementer: author and refine documentation artifacts
- reviewer: test clarity, consistency, and factual grounding
- independent validator: verify claims against source evidence
- release certifier: approve publish-ready docs and caveats

## End-to-End Flow Coverage

This task family maps to complete lifecycle coverage:

1. intake and audience/intention alignment
2. ambiguity resolution for missing source context
3. drafting plan and evidence-backed writing
4. review, validation, and repair loops
5. release certification with publish notes

## Release Expectations

Release must include intended audience, scope boundaries, source-backed claims, and change-summary notes.
