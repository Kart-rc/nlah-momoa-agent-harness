# Generic Documentation Harness

## Purpose

Deliver accurate, audience-aware documentation with traceable sources and usability checks.

## Required Inputs

- documentation objective
- audience and use context
- source materials or system context

## Required Artifacts

- `DOC_OBJECTIVE.md`
- `AUDIENCE_PROFILE.md`
- `SOURCE_INDEX.md`
- `DOC_DRAFT.md`
- `FACT_CHECK_REPORT.md`
- `STYLE_REVIEW.md`
- `RELEASE_PACKET.md`

## Workflow

1. Define objective and audience
2. Collect source material
3. Draft structure and content
4. Fact-check all claims
5. Review clarity, tone, and completeness
6. Revise and finalize

## Validation Gates

- claims are evidence-backed
- terminology is consistent
- instructions are executable
- known gaps are documented
